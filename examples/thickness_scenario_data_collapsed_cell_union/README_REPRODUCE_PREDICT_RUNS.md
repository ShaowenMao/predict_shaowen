# Reproducing the collapsed cell-union PREDICT runs

## Experiment summarized here

This folder contains the completed PREDICT experiment generated by
`examples/gom_perm_varying_thickness_geology_cases_collapsed_cell_union.m`.
The experiment uses:

- 6 stratigraphic thickness/lithology scenarios
- 6 throw windows (`famp1` through `famp6`)
- 27 geology cases: 3 faulting depths x 3 sand Vcl values x 3 clay Vcl values
- 2,000 accepted realizations for every scenario/window/geology combination
- collapsed adjacent layers of the same lithology
- `SmearOverlapRule = 'cell_union_psmear'`

This gives 972 permeability distributions and 1,944,000 accepted realizations.
The completed metadata table contains 972 rows, all result files are present,
and no realization was rejected in this run.

Each `predict_runs.mat` contains `perms`, a 2000-by-3 matrix in mD. Its columns
are `[Kxx, Kyy, Kzz]`.

## Authoritative files

The following files should be shared together. No CSV by itself contains all
PREDICT inputs because the base geometry and stratigraphy of the six throw
windows are defined in the MATLAB runner.

1. `../gom_perm_varying_thickness_geology_cases_collapsed_cell_union.m`
   - Authoritative run driver.
   - Defines the base thickness, burial depth, dip, fault dip, and other
     window-specific inputs.
   - Collapses adjacent equal-lithology layers before constructing each
     `FaultedSection`.
   - Fixes material placement to `cell_union_psmear`.

2. `thickness_scenario_definitions.csv`
   - Human-readable input table with the 36 scenario-window rows.
   - `FWPattern` and `HWPattern` use `S` for sand and `C` for clay.
   - This generated snapshot can be supplied through
     `ThicknessScenarioFile` to make the scenario definitions explicit.

3. `geology_case_definitions.csv`
   - Human-readable audit table for the 27 combinations of faulting depth,
     sand Vcl, and clay Vcl.
   - These cases are generated by the three parameter vectors in the run
     command below; this CSV is not read directly by the runner.

4. `collapsed_thickness_scenario_definitions.csv`
   - Derived audit table showing the original and collapsed S/C patterns.
   - It is an output summary, not a separate simulation input.

5. `collapsed_cell_union_geology_case_definitions.mat`
   - Machine-readable snapshot of the geology table, original scenario table,
     collapsed scenario table, windows, and parsed run options.
   - This is the closest single-file summary, but the MATLAB runner is still
     required for the base window geometry and simulation logic.

6. `tables/collapsed_cell_union_geology_run_metadata.csv`
   - One row per scenario/window/geology run.
   - Records collapsed thicknesses and burial depths, overlap rule, case
     parameters, target sample count, acceptance statistics, and seed base.
   - `CheckpointFile` contains the absolute path on the computer that produced
     the data. Ignore that column on another computer and use the relative
     output structure described below.

Exact replication also requires the complete PREDICT source tree, especially
`functions/faultMaterialMap.m` and `classes/Fault2D.m`, because they implement
`cell_union_psmear`. Use this verified repository snapshot:

- Repository: `https://github.com/ShaowenMao/predict_shaowen`
- Commit: `d417960ffb60a3eb0e655558c01174168352778e`

## Explicit reproduction command

Set `repoRoot`, `mrstRoot`, and `outputDir` for the new computer, then run:

```matlab
repoRoot = 'C:\path\to\predict_shaowen';
mrstRoot = 'C:\path\to\mrst';
outputDir = 'C:\path\to\reproduced_thickness_scenario_data';

run(fullfile(mrstRoot, 'startup.m'));
addpath(fullfile(repoRoot, 'examples'));

scenarioFile = fullfile(repoRoot, 'examples', ...
    'thickness_scenario_data_collapsed_cell_union', ...
    'thickness_scenario_definitions.csv');

results = gom_perm_varying_thickness_geology_cases_collapsed_cell_union( ...
    outputDir, ...
    'Windows', {'famp1', 'famp2', 'famp3', 'famp4', 'famp5', 'famp6'}, ...
    'Nsim', 2000, ...
    'FaultingDepths', [50 500 1000], ...
    'SandVclValues', [0.1 0.2 0.3], ...
    'ClayVclValues', [0.4 0.5 0.6], ...
    'IsClayVcl', 0.4, ...
    'CorrCoef', 0.6, ...
    'BaseSeed', 1729, ...
    'ThicknessScenarioFile', scenarioFile, ...
    'ThicknessScenarios', 1:6, ...
    'SmearOverlapRule', 'cell_union_psmear', ...
    'BatchSize', 200, ...
    'UseParallel', true, ...
    'NumWorkers', 16, ...
    'Resume', true, ...
    'ShowProgress', true);
```

`UseParallel` and `NumWorkers` affect performance, not the intended random
sequence, because each attempted realization receives its own deterministic
seed. The original run used 16 workers.

## Seed and replay information

The base seed was 1729. A scenario/window/case seed base is calculated as:

```matlab
seedBase = 1729 + 100000000*scenarioIndex ...
                + 10000000*windowIndex ...
                + 100000*caseIndex;
```

Every completed `predict_runs.mat` stores:

- `meta.SeedBase`
- `meta.AcceptedSeeds`
- `meta.AcceptedAttemptIndices`
- `meta.NumAttempts`
- `meta.NumRejected`
- `checkpointInfo`, containing the complete case identity

For exact replay of accepted realization `i`, use
`meta.AcceptedSeeds(i)`. Do not infer an accepted seed only from its row number
in a dataset where rejected attempts may occur.

## Output structure

```text
outputDir/
  geology_case_definitions.csv
  thickness_scenario_definitions.csv
  collapsed_thickness_scenario_definitions.csv
  collapsed_cell_union_geology_case_definitions.mat
  tables/
    collapsed_cell_union_geology_run_metadata.csv
  data/<scenarioLabel>/<window>/<caseLabel>/
    predict_runs.mat
```

An incomplete run may also contain `predict_runs.progress.mat`. Re-running the
same command with `Resume = true` continues compatible partial cases and skips
completed cases.

## Software requirements and reproducibility limitation

The code requires MATLAB, the Statistics and Machine Learning Toolbox, and
MRST. Parallel Computing Toolbox is required only for `UseParallel = true`.
The runner loads the MRST modules `mrst-gui`, `coarsegrid`, `upscaling`,
`incomp`, `mpfa`, and `mimetic`, and uses TPFA upscaling.

The checkpoint files do not record the MATLAB release, MRST release, operating
system, or MEX build information. An earlier console log in the project showed
MRST 2024b, but that does not prove which MRST release generated this completed
collapsed dataset. Confirm and record the actual MATLAB and MRST versions before
claiming bit-for-bit reproducibility. Using the same inputs, seeds, source
commit, and compatible software should reproduce the same distributions, but
different solver versions can introduce small numerical differences.
